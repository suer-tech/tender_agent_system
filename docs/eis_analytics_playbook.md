# ЕИС / ДИС — карта данных и сценарии аналитики

Составлено 2026-04-21 по справочнику `nsiDocumentTypes` / `nsiDocumentTypes223`, вытащенному через `getNsi`. Полный машинный каталог — в `diag/eis_nsi_doctypes_catalog.txt` (44-ФЗ 126 публичных типов × 18 подсистем, 223-ФЗ 93 типа × 13 подсистем).

## Как читать этот документ
- **Подсистема** (`subsystemType` в SOAP-запросе) = смысловой контур ЕИС. Аналитика всегда идёт через сквозную связку 2–4 подсистем.
- **Тип документа** (`documentType44/223`) = конкретный XSD-документ внутри подсистемы. При запросе к ДИС указываются ОБА.
- «Публичный» = `isDistributedPublicly=true` — получаемый по нашему токену без спец-доступа. Непубличные (отчёты Казначейства, БО/МО, закрытые контракты) — недоступны.

## Временной охват и лимиты

- **44-ФЗ:** с 2014 г. (ввод ЕИС). Под 94-ФЗ (2005–2013) — частичное покрытие.
- **223-ФЗ:** с 2012, плотно — с 2015.
- Гранулярность: **по суткам** (`exactDate` YYYY-MM-DD). Также доступны `startDate/endDate` и почасовая выборка для плотных дней.
- «Полный» день лучше забирать с лагом **1–2 суток** — данные дописываются в течение дня.
- 223-ФЗ на общем endpoint `int.zakupki.gov.ru` возвращает echo запроса — вероятно требует отдельного канала (`int223` + ГОСТ-TLS). Реальная выгрузка 223-ФЗ у нас пока не работает.

## Сценарии аналитики (что точно можно построить)

### A. Продукты для поставщика (лидогенерация и КП)

1. **Лента свежих торгов под профиль поставщика.** PRIZ/`epNotification*` + RI223/`purchaseNotice*` → фильтр по ОКПД2, региону, НМЦК, способу закупки. Уже есть в боте, EIS даёт **историю** и **бэкап** к парсингу площадок.
2. **Ценовые бенчмарки для КП.** PRIZ + RGK.`contract` за 12–24 мес.: «на эту ОКПД2 в этом регионе прошло 120 закупок, стартовая цена медиана X, финальная Y, средняя скидка Z%».
3. **Предсказание предстоящих торгов.** RPGZ/`tenderPlan2020` + RPZ223/`purchasePlan`: «заказчик в Q3 планирует закупить X на Y ₽». Опережение конкурентов на 1–2 квартала.
4. **Паттерны «типичного победителя».** PRIZ финальные протоколы + RGK: связка (заказчик × ОКПД2) — кто чаще побеждает, с какой скидкой, с какой скоростью подаёт первую цену. Определение «прикормленный vs открытый» тендер.

### B. Продукты для due diligence (проверка контрагента)

5. **Поставщик-досье.** По ИНН → RGK + RD223 + UR + RNP/RNP223 + RRK. Результат: «исполнено 127 контрактов на 230 млн, 2 расторжения по инициативе заказчика, нет в РНП». Монетизируется как B2B-отчёт.
6. **Заказчик-досье.** По ИНН/КПП → структура закупок, частые ОКПД2, средний чек, доля у МСП (OZ/`fcsCustomerReportSmallScaleBusiness`), доля отечественного (`...RusProductsPurchasesVolume`), история жалоб.
7. **Красные флаги.** RJ/RJ223 + `tenderSuspension` + RVP + UR + RNP. Риск-скор для каждого заказчика/поставщика.

### C. Отраслевая и макро-аналитика (B2B/консультанты)

8. **Карта рынка по ОКПД2.** Объёмы госзакупок по (ОКПД2 × регион × квартал) за 5–10 лет — прямые цифры без выборок. То, что ВШЭ и Аналитический центр продают как отчёты.
9. **Концентрация поставщиков (HHI).** По (ОКПД2 × регион): сколько игроков реально конкурирует, где монополия, где открытый рынок.
10. **Сезонность и бюджетные пики.** По RGK: ноябрь–декабрь — «распиловка остатков», апрель–май — провал. Количественно по отраслям.
11. **Доля МСП и отечественного.** OZ отчёты + MSP223 для 223-ФЗ. Соблюдают ли квоты, у кого провал.

### D. Специфичные ниши

12. **ЖКХ / капремонт.** PPRF615 + RD615 + RKPO — рынок капремонта многоквартирных домов.
13. **Строительство и исполнение.** EA/`ON_AKTREZRAB*` (КС-2/КС-3) + УПД — этапы «деньги ↔ объём».
14. **223-ФЗ МСП-квоты.** RI223/…SMBO* (суффикс SMBO) — отдельный срез закупок для малого бизнеса у госкорпораций.

### E. Сигналы злоупотреблений

15. **Повторяющиеся пары.** RGK: связка «заказчик → один поставщик → N контрактов подряд с высокой долей». Признак аффилированности.
16. **Подозрительные скидки.** Скидка <1% при НМЦК >5 млн + единственный участник (виден в `epProtocolEF2020Final` / `purchaseProtocolRZAE`).
17. **Каскад жалоб на заказчика.** RJ за период → несколько обоснованных → на карандаш.

## Приоритизация (что запускать первым)

- **Спринт 1 (1–2 нед.):** PRIZ + RGK за 24 мес., ценовые бенчмарки по ОКПД2 × регион. Самая денежная фича для поставщиков, прямой апгрейд к боту.
- **Спринт 2:** RNP + UR + RJ — риск-скор для заказчиков/поставщиков, карточки в веб-UI.
- **Спринт 3:** RPGZ — плановые закупки, опережение конкурентов. Уникальное конкурентное преимущество.
- **Далее:** отраслевые дашборды (карта рынка), ЖКХ/стройка, 223-ФЗ (после решения канала).

### Ограничения
- **223-ФЗ через ДИС пока не работает** (echo-респонс на общем endpoint). Нужен `int223` + ГОСТ-TLS либо альтернативный канал.
- **Объёмы:** 85 регионов × ~20 актуальных типов × 730 дней (2 года) = ~1.2 млн запросов при полной выгрузке. Нужна очередь + уважение лимитов ДИС (точная цифра не публикуется). Начинать с 2–3 массовых типов + Москва/СПб/МО.
- **Коммерческие B2B-площадки** (Фабрикант, B2B-Center, Росэлторг) ДИС не покрывает — остаются на отдельных парсерах.
